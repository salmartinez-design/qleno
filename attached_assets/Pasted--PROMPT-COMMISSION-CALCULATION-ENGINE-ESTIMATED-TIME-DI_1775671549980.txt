**STEP 2 — ESTIMATED HOURS AND LABOR SPLITTING**

When a job is created (from booking widget or internally), store the total estimated hours from the pricing engine tier lookup into `jobs.estimated_hours`. This is the total labor hours for the job regardless of how many techs are assigned.

**`estimated_hours_per_tech` calculation:**

This field must recalculate automatically whenever the number of assigned techs changes:

```
estimated_hours_per_tech = estimated_hours ÷ number_of_techs_assigned
```

Rules:
- 1 tech assigned: `estimated_hours_per_tech = estimated_hours` (e.g. 6.2 hrs)
- 2 techs assigned: `estimated_hours_per_tech = estimated_hours ÷ 2` (e.g. 3.1 hrs)
- 3 techs assigned: `estimated_hours_per_tech = estimated_hours ÷ 3` (e.g. 2.07 hrs)
- If job moves from assigned back to unassigned: `estimated_hours_per_tech = estimated_hours` (reset to full hours)
- Round to 2 decimal places

Trigger this recalculation whenever:
- A tech is added to the job
- A tech is removed from the job
- Job status changes to unassigned

**This is the time shown to technicians on their My Jobs view and on the dispatch board — not the total job hours.**

**STEP 3 — COMMISSION CALCULATION FUNCTION**

Create server-side function `calculateTechPay(jobId)`. Run whenever a tech is assigned, removed, clocks in/out, or commission rate is changed.

**RESIDENTIAL JOBS (`job_type = 'residential'`):**

Step A — Commission pool:
```
commission_pool = job_total × (commission_rate ÷ 100)
```

Step B — Minutes worked per tech:
- Read `clock_in_time` and `clock_out_time` from existing timeclock/job_technicians clock data
- `minutes_worked = clock_out_time - clock_in_time` for each tech
- Use the last recorded clock-out on the job as end time for all techs
- If no clock data yet (job not started): use equal split — `minutes_worked = estimated_hours_per_tech × 60` for each tech

Step C — Proportional split:
```
total_minutes = sum of all techs' minutes_worked
each_tech_pay = (their_minutes ÷ total_minutes) × commission_pool
```

Round to 2 decimal places. Assign rounding remainder to the tech with the most minutes.

**Verified examples:**
- 1 tech, $180 job, 35% → $63.00
- 1 tech, $490 job, 35% → $171.50
- 2 techs same clock-in, $240 job, 35% → $42.00 each
- 2 techs, Tech A 360 min, Tech B 350 min, $240 job, 35% pool $84 → Tech A $42.59, Tech B $41.41
- 3 techs equal time, $741.60 job, 35% → $86.54 each (11.67% each)

**COMMERCIAL JOBS (`job_type = 'commercial'`):**

```
each_tech_pay = hourly_rate × actual_hours_worked
```

- Default `hourly_rate = 20.00` per tech
- Estimated pay = `hourly_rate × estimated_hours_per_tech` until clock-out
- Finalized at clock-out using actual hours
- Office can override `hourly_rate` per tech per job

**`final_pay` logic:**
- If `override_pay` is set → `final_pay = override_pay`
- Otherwise → `final_pay = calculated_pay`
