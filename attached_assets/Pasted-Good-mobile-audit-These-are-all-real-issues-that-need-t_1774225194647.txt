FIX 1 — Revenue Summary: Week and Month buttons broken

On the Revenue Summary report at /reports/revenue, the 
"Week" and "Month" group-by buttons do nothing when tapped 
on mobile. Only "Day" works. Find the button handler for 
the group-by selector and fix it so all three options 
(Day, Week, Month) correctly update the report data and 
re-render the chart on mobile.

---

FIX 2 — Clock Monitor: visual polish

The Clock Monitor page has a functional layout but the 
4 stat cards (Total Entries, Currently Active, Flagged, 
Out of Range) use colored backgrounds that feel inconsistent 
with the design system. 

Update the stat cards to match the design system:
- All 4 cards: white background (#FFFFFF), 
  #E5E2DC border
- Currently Active number: use green text (#16a34a)
- Flagged number: use red text (#dc2626)  
- Out of Range number: use amber text (#d97706)
- Total Entries number: use primary text (#1A1917)
- Consistent card height and padding across all 4
- "No clock entries today" empty state: center it 
  properly with more vertical breathing room

FIX 3 — Performance Insights: horizontal scroll

The Performance Insights report page has content 
overflowing horizontally on mobile causing unwanted 
side-scroll. 

Fix by:
- The two-column card grid (Top Performers / Employees 
  Needing Attention) should stack to single column 
  on mobile (max-width: 640px)
- The Revenue by Service Type section should be 
  full width on mobile
- No element should exceed the viewport width
- Add overflow-x: hidden to the page container
- All cards should be 100% width on mobile with 
  16px horizontal padding

FIX 4 — Company Settings: horizontal scroll

Same issue as Fix 3. The Company Settings tab bar 
(General, Branding, Notifications, Clock In/Out, 
Invoicing, etc.) overflows horizontally on mobile.

Fix the tab bar so it scrolls horizontally within 
the tab bar itself (overflow-x: auto with 
scrollbar hidden) but does not cause the entire 
page to scroll sideways. The tab content below 
should always be 100% viewport width.

Also check all form fields inside each settings tab — 
any two-column form layouts should collapse to 
single column on mobile.

The Rate Increase tool appears in the mobile nav

FIX 6 — New Job wizard: add new customer from mobile

When creating a new job on mobile via the dispatch 
board, the client search step only allows searching 
existing clients. There is no way to add a new 
customer inline.

Add a "New Customer" option to the client search step:
- Below the search results, show a "Add New Customer" 
  button
- Tapping it opens a compact inline form with fields:
  First name, Last name, Phone, Email, Address
- On save, creates the customer record and 
  auto-selects them for the job
- Returns to the job wizard with the new customer 
  selected
- This must work fully on mobile without navigating 
  away from the job wizard

---

FIX 7 — New Job wizard: quote from mobile

Add a "Get Quote" option in the job wizard details 
step on mobile. When a service type is selected 
and duration is set, show a "View Quote" summary 
section that displays:
- Service type
- Duration
- Calculated price (using the pricing engine — 
  call POST /api/pricing/calculate with the 
  selected scope and duration)
- Frequency
- A "Send Quote to Client" button that creates 
  a quote record and sends it via email 
  (requires Resend — show disabled state with 
  tooltip "Email not configured" if RESEND_API_KEY 
  is not set)

This gives staff the same quoting capability 
on mobile as the desktop quote builder.

FIX 8 — Reschedule on mobile job panel

The job detail panel (bottom sheet) on mobile 
currently shows: Mark Complete, Start Job, Flag, 
Cancel Job.

Add a Reschedule button to this panel:
- Place it next to Cancel Job at the bottom
- Tapping it opens a date/time picker sheet
- User selects new date and time
- On confirm: updates the job's scheduled_date 
  and scheduled_time, changes status back to 
  'scheduled' if it was cancelled, logs a 
  cancellation/reschedule record with reason 
  'Rescheduled' in cancellationLogTable
- Show a success confirmation: 
  "Job rescheduled to [new date/time]"
- This must work on mobile and desktop
