You are working on Qleno, a multi-tenant SaaS platform for 
residential and commercial cleaning businesses.

This sprint builds the Commercial Account Architecture — 
the foundational data model that supports property management 
accounts with multiple properties, hourly billing, service-type 
rate cards, multi-contact routing, and consolidated invoicing.

This is a PHES Day 1 requirement. PHES is 70% commercial — 
property management turnovers and common areas — and cannot 
migrate without this architecture in place.

TECH STACK: React 18, Tailwind, Express/Node.js, Supabase 
(Postgres + RLS), Plus Jakarta Sans only.

DESIGN SYSTEM — NON-NEGOTIABLE:
- Background: #F7F6F3
- Cards: #FFFFFF
- Primary text: #1A1917
- Secondary text: #6B6860
- Borders: #E5E2DC
- Accent: var(--brand)
- NO emojis, NO dark mode, light theme only
- Plus Jakarta Sans only
- Skeleton loaders for all async loads
- All monetary values: USD, 2 decimal places, comma-formatted

All endpoints: JWT protected, company_id scoped, RLS enforced.
Never expose data across tenant boundaries.

PART 2 — DATABASE SCHEMA
═══════════════════════════════════════════════════════

Create accounts table:
{
  id                    uuid primary key default gen_random_uuid(),
  company_id            integer not null FK companies,
  account_name          text not null,
  account_type          enum: property_management /
                              commercial_office /
                              retail /
                              other
                        default: property_management,
  invoice_frequency     enum: per_job / weekly / 
                              monthly / custom
                        default: per_job,
  payment_terms_days    integer default 0,
  payment_method        enum: card_on_file / check / 
                              ach / invoice_only
                        default: invoice_only,
  notes                 text,
  is_active             boolean default true,
  created_at            timestamp default now(),
  updated_at            timestamp default now()
}

Create account_rate_cards table:
{
  id                uuid primary key default gen_random_uuid(),
  account_id        uuid not null FK accounts,
  company_id        integer not null FK companies,
  service_type      text not null,
  -- Examples: 'Apartment Turnover', 'Common Areas',
  --   'Deep Clean', 'Move In/Out', 'Standard Clean'
  billing_method    enum: hourly / flat_rate / per_unit
                    default: hourly,
  rate_amount       decimal(10,2) not null,
  unit_label        text default 'hr',
  -- 'hr', 'job', 'unit', 'sq ft'
  is_active         boolean default true,
  notes             text,
  created_at        timestamp default now()
}

Create account_properties table:
{
  id                uuid primary key default gen_random_uuid(),
  account_id        uuid not null FK accounts,
  company_id        integer not null FK companies,
  property_name     text,
  -- e.g. "PPM | 1000 North La Salle"
  address           text not null,
  city              text,
  state             text,
  zip               text,
  unit_count        integer,
  -- Number of units in building (for apartment buildings)
  property_type     enum: apartment_building /
                          condo /
                          common_area /
                          office /
                          retail /
                          other
                    default: apartment_building,
  lat               decimal(10,8),
  lng               decimal(11,8),
  zone_id           uuid nullable FK service_zones,
  default_service_type text,
  -- Pre-fills service type when scheduling a job 
  -- at this property
  access_notes      text,
  -- Gate codes, key location, building access info
  is_active         boolean default true,
  created_at        timestamp default now(),
  updated_at        timestamp default now()
}

Create account_contacts table:
{
  id                    uuid primary key default gen_random_uuid(),
  account_id            uuid not null FK accounts,
  company_id            integer not null FK companies,
  name                  text not null,
  role                  enum: billing / operations /
                              onsite / accountant /
                              property_manager / other
                        default: operations,
  email                 text,
  phone                 text,
  receives_invoices     boolean default false,
  receives_on_way_sms   boolean default false,
  receives_completion_notifications boolean default false,
  is_primary            boolean default false,
  notes                 text,
  created_at            timestamp default now()
}

Modify jobs table — add these fields if not present:
  account_id            uuid nullable FK accounts,
  property_id           uuid nullable FK account_properties,
  billing_method        enum: hourly / flat_rate / per_unit,
  hourly_rate           decimal(10,2),
  -- Pulled from account_rate_cards on job creation
  -- Dispatcher can override
  billed_hours          decimal(5,2),
  -- Auto-calculated from clock in/out
  -- Can be manually overridden before invoicing
  billed_amount         decimal(10,2),
  -- Auto-calculated: billed_hours × hourly_rate
  -- Or flat_rate amount
  service_type          text,
  -- Drives rate card lookup

PART 3 — HOURLY BILLING ENGINE
═══════════════════════════════════════════════════════

When a job has billing_method = 'hourly':

ON JOB COMPLETION:
  1. Pull all timeclock entries for this job_id
  2. Calculate total_hours:
     SUM of (clock_out - clock_in) across all 
     timeclock entries for this job
     Round to 2 decimal places
     (e.g. 2 hours 37 minutes = 2.62 hours)
  3. Set jobs.billed_hours = total_hours
     (can be manually edited by owner/admin 
      before invoice is created)
  4. Set jobs.billed_amount = 
     billed_hours × jobs.hourly_rate
  5. Auto-create invoice with:
     line_items: [{
       description: service_type + ' — ' + 
                    property_name + ' · ' + 
                    billed_hours + ' hrs @ $' + 
                    hourly_rate + '/hr',
       quantity: billed_hours,
       unit_price: hourly_rate,
       total: billed_amount
     }]
     total_amount: billed_amount
     due_date: today + account.payment_terms_days

MANUAL HOURS OVERRIDE:
  Owner/admin can edit billed_hours on a completed 
  job before the invoice is sent.
  Editing billed_hours auto-recalculates billed_amount.
  Show original clock-in/out derived hours alongside 
  the editable field:
  "Actual time on site: 2.62 hrs · Billed: [   ] hrs"

For flat_rate jobs:
  billed_amount = jobs.price (existing field)
  No hours calculation needed

═══════════════════════════════════════════════════════
PART 4 — RATE CARD AUTO-POPULATION
═══════════════════════════════════════════════════════

When creating a job:

  If account_id is set AND service_type is selected:
    1. Query account_rate_cards WHERE
       account_id = selected account AND
       service_type = selected service_type AND
       is_active = true
    2. If rate found:
       Auto-fill billing_method and hourly_rate
       Show read-only indicator: 
       "Rate from account card: $60/hr"
       Allow dispatcher to override with note
    3. If no rate found for this service type:
       Show warning: 
       "No rate configured for this service type.
        Enter rate manually or update the 
        account rate card."
       Allow manual entry

  If no account_id (residential or unlinked job):
    Use existing flat-rate pricing flow

PART 5 — CONSOLIDATED INVOICING
═══════════════════════════════════════════════════════

For accounts with invoice_frequency = 'weekly' 
or 'monthly':

  Do NOT auto-create per-job invoices.
  Instead: mark job as 'pending_consolidation' 
  after completion.

  Add POST /api/accounts/:id/generate-invoice
  — Owner/admin triggers manually OR 
    runs on schedule based on invoice_frequency
  — Collects all jobs for this account with 
    status = 'complete' AND invoice_id IS NULL
    within the billing period
  — Creates ONE invoice with multiple line items:
    One line item per job:
    { description: date + property + service_type + 
                   hours + rate,
      quantity: billed_hours,
      unit_price: hourly_rate,
      total: billed_amount }
  — Invoice subtotal = sum of all line items
  — Due date = invoice date + payment_terms_days

  On the account profile:
    "Generate Invoice" button visible when there 
    are uninvoiced completed jobs.
    Shows count: "4 jobs pending invoicing 
    — total $847.00"
    Clicking opens a preview of the consolidated 
    invoice before confirming.

         property_count, active_properties_count,

ACCOUNT PROPERTIES

       — soft delete only

ACCOUNT CONTACTS
GET    /api/accounts/:id/contacts

CONSOLIDATED INVOICING

       — preview or confirm consolidated invoice

       — list all uninvoiced completed jobs

RATE CARD LOOKUP
GET    /api/accounts/:id/rates/lookup?service_type=
       — returns matching rate for job creation
         auto-population

client_type = commercial OR when account_id is set:

    Amber warning: "No rate for this service type.
    Set manually or add to account rate card."
    Manual rate entry field appears.

Step 3 — Schedule & Team:
  Existing smart dispatch flow unchanged.
  Property address used for proximity calculation.

Step 4 — Confirmation:
  Shows: Account name, property, service type,
  billing method, rate, scheduled time, team.
  For hourly jobs:
  "Invoice will be calculated from actual 
   clock in/out time after job completion."
