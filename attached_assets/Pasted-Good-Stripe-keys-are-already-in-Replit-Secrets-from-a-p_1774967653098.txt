**REPLIT PROMPT — AUTOMATED FOLLOW-UP ENGINE:**

Build a native follow-up sequence engine with the following database tables, cron logic, and settings UI.

**STEP 1 — DATABASE TABLES**

Create these tables:

```sql
-- Sequence definitions per company
CREATE TABLE follow_up_sequences (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  company_id uuid REFERENCES companies(id),
  sequence_type TEXT NOT NULL, 
  -- 'quote_followup' | 'post_job_retention'
  name TEXT NOT NULL,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Individual steps within a sequence
CREATE TABLE follow_up_steps (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  sequence_id uuid REFERENCES follow_up_sequences(id),
  step_number INTEGER NOT NULL,
  delay_hours INTEGER NOT NULL,
  channel TEXT NOT NULL, -- 'sms' | 'email'
  subject TEXT, -- email only
  message_template TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Active enrollments per lead or client
CREATE TABLE follow_up_enrollments (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  company_id uuid REFERENCES companies(id),
  sequence_id uuid REFERENCES follow_up_sequences(id),
  lead_id uuid REFERENCES leads(id) NULL,
  client_id uuid REFERENCES clients(id) NULL,
  current_step INTEGER DEFAULT 1,
  enrolled_at TIMESTAMPTZ DEFAULT now(),
  next_fire_at TIMESTAMPTZ NOT NULL,
  completed_at TIMESTAMPTZ NULL,
  stopped_at TIMESTAMPTZ NULL,
  stopped_reason TEXT NULL
  -- 'booked' | 'closed' | 'rebooked' | 'max_steps'
);

-- Log of every message sent
CREATE TABLE message_log (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  company_id uuid REFERENCES companies(id),
  enrollment_id uuid REFERENCES follow_up_enrollments(id),
  channel TEXT NOT NULL,
  recipient_phone TEXT NULL,
  recipient_email TEXT NULL,
  subject TEXT NULL,
  body TEXT NOT NULL,
  status TEXT NOT NULL, -- 'sent' | 'failed'
  sent_at TIMESTAMPTZ DEFAULT now()
);
```

**STEP 4 — CRON JOB**

Add to the existing nightly cron (or create a separate cron running every 30 minutes):

```
SELECT all follow_up_enrollments WHERE 
  completed_at IS NULL 
  AND stopped_at IS NULL 
  AND next_fire_at <= NOW()

For each enrollment:
  1. Fetch the current step from follow_up_steps
  2. Resolve merge fields from the linked lead or client record
  3. Send via Twilio (sms) or Resend (email)
  4. Insert a row into message_log
  5. If next step exists: increment current_step, 
     set next_fire_at = NOW() + next step delay
  6. If no next step: set completed_at = NOW()
```

**STOP CONDITIONS — wire these:**

- Lead status changes to 'booked' or 'closed' → set stopped_at = NOW(), stopped_reason = 'booked' or 'closed' on any active quote_followup enrollment for that lead
- Job is created for a client with active post_job_retention enrollment → set stopped_at = NOW(), stopped_reason = 'rebooked'
- Recurring clients (clients with a recurring_schedule record and a future job) → never enroll in post_job_retention
