**PART 5 — DATABASE MIGRATIONS**

Run all of the following:

```sql
-- Jobs table additions
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS upsell_shown BOOLEAN DEFAULT false;
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS upsell_accepted BOOLEAN DEFAULT false;
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS upsell_declined BOOLEAN DEFAULT false;
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS upsell_deferred BOOLEAN DEFAULT false;
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS upsell_cadence_selected VARCHAR(20);
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS property_vacant BOOLEAN DEFAULT false;
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS first_recurring_discounted BOOLEAN DEFAULT false;
ALTER TABLE jobs DROP COLUMN IF EXISTS move_date;

-- Rate locks table
CREATE TABLE IF NOT EXISTS rate_locks (
  id SERIAL PRIMARY KEY,
  company_id INTEGER NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  client_id INTEGER NOT NULL REFERENCES clients(id) ON DELETE CASCADE,
  recurring_schedule_id INTEGER REFERENCES recurring_schedules(id) ON DELETE SET NULL,
  locked_rate NUMERIC(10,2) NOT NULL,
  cadence VARCHAR(20) NOT NULL,
  lock_start_date DATE NOT NULL,
  lock_expires_at DATE NOT NULL,
  active BOOLEAN NOT NULL DEFAULT true,
  renewal_alert_30_sent BOOLEAN DEFAULT false,
  created_manually BOOLEAN DEFAULT false,
  voided_at TIMESTAMP,
  void_reason VARCHAR(50),
  void_notes TEXT,
  voided_by_user_id INTEGER,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_rate_locks_client ON rate_locks(client_id);
CREATE INDEX IF NOT EXISTS idx_rate_locks_active ON rate_locks(active, lock_expires_at);

-- Offer settings table
CREATE TABLE IF NOT EXISTS offer_settings (
  id SERIAL PRIMARY KEY,
  company_id INTEGER NOT NULL UNIQUE REFERENCES companies(id) ON DELETE CASCADE,
  upsell_enabled BOOLEAN NOT NULL DEFAULT true,
  upsell_discount_percent NUMERIC(5,2) NOT NULL DEFAULT 15.00,
  rate_lock_enabled BOOLEAN NOT NULL DEFAULT true,
  rate_lock_duration_months INTEGER NOT NULL DEFAULT 24,
  overrun_threshold_percent NUMERIC(5,2) NOT NULL DEFAULT 20.00,
  overrun_jobs_trigger INTEGER NOT NULL DEFAULT 2,
  service_gap_days INTEGER NOT NULL DEFAULT 60,
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Seed defaults for all existing companies
INSERT INTO offer_settings (company_id)
SELECT id FROM companies
ON CONFLICT (company_id) DO NOTHING;
```

**On upsell accepted — form submission flow:**
1. Create Deep Clean job with all upsell tracking columns set correctly
2. Create `recurring_schedules` record with selected cadence, client_id, company_id, service_address, scope
3. Create first recurring job tagged `first_recurring_discounted: true` — price at full rate × (1 - upsell_discount_percent/100) pulled from offer_settings
4. Insert `rate_locks` record if `rate_lock_enabled = true`:
   - `locked_rate`: full recurring rate (not discounted first-visit rate)
   - `cadence`: selected cadence
   - `lock_start_date`: today
   - `lock_expires_at`: today + rate_lock_duration_months from offer_settings
   - `active`: true

**First recurring discount expiry protection:**
Before generating or charging any recurring job, check if any prior completed job for that recurring_schedule_id has `first_recurring_discounted: true`. If yes — charge full `locked_rate` on all subsequent jobs. Never apply first-visit discount more than once per recurring schedule.

**PART 6 — RATE LOCK AUTO-VOID + RENEWAL LOGIC**

Create `server/utils/rateLock.ts`.

**checkRateLockVoidConditions(clientId, companyId):**
Call this after every job is marked complete.

```
Pull offer_settings for company
Query last 3 completed recurring jobs for this client

Condition 1 — Time overrun:
Count jobs where actual_duration > estimated_duration * (1 + overrun_threshold_percent/100)
If count >= overrun_jobs_trigger:
  Set rate_lock active = false
  void_reason = 'time_overrun'
  voided_at = NOW()
  Send Resend email to info@phes.io:
    Subject: "Rate Lock Voided — Time Overrun: [Client Name]"
    Body: client name, locked rate, actual vs estimated duration average, prompt to re-quote
```

**runRateLockNightlyChecks():**
Call alongside recurring engine in nightly cron.

```
Pull all active rate_locks

Condition 2 — Service gap:
Pull offer_settings.service_gap_days for each lock's company
Find most recent completed job for each client
If most_recent_job_date < TODAY - service_gap_days:
  Set active = false, void_reason = 'service_gap', voided_at = NOW()
  Send Resend email to info@phes.io:
    Subject: "Rate Lock Voided — Service Gap: [Client Name]"
    Body: client name, last service date, days since service

Condition 3 — Natural expiry:
If lock_expires_at < TODAY and active = true:
  Set active = false, void_reason = 'expired'
  No email

Condition 4 — Renewal alert:
If lock_expires_at = TODAY + 30 days and renewal_alert_30_sent = false:
  Send Resend email to info@phes.io:
    Subject: "Rate Lock Expiring Soon: [Client Name]"
    Body: client name, locked rate, cadence, expiry date
    Prompt: "Consider reaching out to offer a renewal before expiry."
  Set renewal_alert_30_sent = true
```
