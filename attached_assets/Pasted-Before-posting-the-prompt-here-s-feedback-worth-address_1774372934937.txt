Before posting the prompt, here's feedback worth addressing first:

---

**Issues spotted in the MC data:**

1. **Duplicate Promo Discount on Hourly Standard** — one at -10% and one at -15%. Which is correct? Decide before seeding or you'll have the same problem in Qleno.

2. **$0.00 extras on all hourly scopes** — Oven, Refrigerator, Kitchen Cabinets, Windows, Basement are all $0.00 on hourly scopes. This is intentional in MC because hourly billing already captures the time — but it looks broken in a UI. Better pattern: hide $0.00 price extras from the client-facing quote and label them "Time add-on only" internally.

3. **Windows pricing inconsistency across scopes** — 15% sqft on Deep Clean, 12% each on Standard, 12% sqft on Recurring. These are different calculation methods for the same service. Confirm these are intentional before seeding.

4. **Loyalty Discount duplicated at two tiers ($100 / $50)** across nearly every scope — this is MC's workaround for not having tiered loyalty. Qleno should handle this cleaner with a single loyalty discount tied to the client's loyalty tier from the loyalty system already built.

5. **Adjustment +/- at $1.00 each** — this is a manual override tool. $1.00 increments are tedious. Better pattern: a free-form dollar input field labeled "Manual Adjustment" with +/- toggle.

---

**Structural improvement for Qleno vs MC:**

MC duplicates the same add-ons across every scope (Oven $50, Refrigerator $50, Parking $20 appear 5+ times each). This creates maintenance overhead — if you raise the oven fee you have to update it in 5 places.

Qleno should use a **shared add-on library** with scope assignment flags instead.
