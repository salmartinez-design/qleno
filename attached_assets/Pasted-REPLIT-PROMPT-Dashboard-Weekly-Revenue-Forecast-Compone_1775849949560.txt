It consists of three week cards stacked vertically: Last Week → Current Week → Next Week.WEEK DEFINITIONPhes week runs Sunday through Saturday. For any given render date (today):

Last week: The most recent completed Sun–Sat range (all days in the past)
Current week: The Sun–Sat range containing today
Next week: The Sun–Sat range immediately following current week
Sunday and Saturday always show $0 / 0 jobs with a closed state — Phes does not operate on weekends. Never query or display revenue for Sun/Sat.DATA QUERIES — one API endpoint: GET /api/dashboard/weekly-forecastReturns three week objects. Each week object contains 7 day entries (Sun–Sat). Build this endpoint in the Express backend.Last week — actual data:
For each Mon–Fri in the last week range, query:

revenue: SUM of amount from job_history WHERE company_id = current AND service_date = that day
job_count: COUNT of rows from job_history WHERE company_id = current AND service_date = that day
type: 'actual'
Current week — booked data:
For each Mon–Fri in the current week range:

For days in the past (before today): same query as last week against job_history
For today and future days: query jobs table WHERE company_id = current AND scheduled_date = that day AND status NOT IN ('void', 'moved', 'skip', 'cancelled')
revenue: SUM of price (or estimated_revenue — use whatever revenue field exists on the jobs table)
job_count: COUNT of those jobs
unassigned_count: COUNT WHERE additionally assigned_technician_id IS NULL
type: 'current'
Next week — projected data:
For each Mon–Fri in next week range, query jobs table same as current week future days.

type: 'projected'
Daily average — compute server-side: AVG of Mon–Fri revenue from job_history for the past 8 weeks (excluding current and next week). Return this as daily_avg on each week object. Used for color thresholds on the frontend.COLOR THRESHOLDS — per day cell (Mon–Fri only)Using daily_avg from the API response:ConditionStylerevenue >= daily_avg * 0.9Green: bg #EAF3DE, revenue text #27500A, jobs text #3B6D11, bar #639922revenue >= daily_avg * 0.6Amber: bg #FAEEDA, revenue text #633806, jobs text #854F0B, bar #EF9F27revenue < daily_avg * 0.6 AND revenue > 0Red: bg #FCEBEB, revenue text #791F1F, jobs text #A32D2D, bar #E24B4Arevenue = 0 AND day is pastRed: same as aboverevenue = 0 AND day is future (projected)Amber with dashed border 1.5px dashed #E5E2DCFor type: 'projected' days, override background to #F7F6F3 with border: 0.5px dashed #E5E2DC UNLESS the day is red (critically thin) — red projected days use border: 1.5px dashed #E24B4A with red text to surface urgency early.Sun/Sat: always background: #F7F6F3, text #6B6860, bar background: #E5E2DC.
