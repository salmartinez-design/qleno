NEW DB TABLES TO CREATE
═══════════════════════════════════════════════════════════

loyalty_tiers (
  id SERIAL PRIMARY KEY,
  company_id UUID REFERENCES companies(id),
  tier_name TEXT NOT NULL,
  min_visits INTEGER DEFAULT 0,
  min_lifetime_revenue NUMERIC DEFAULT 0,
  reward_description TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
)

client_loyalty (
  id SERIAL PRIMARY KEY,
  client_id UUID REFERENCES clients(id),
  company_id UUID REFERENCES companies(id),
  tier_id INTEGER REFERENCES loyalty_tiers(id),
  tier_override TEXT,
  points_balance INTEGER DEFAULT 0,
  total_points_earned INTEGER DEFAULT 0,
  notes TEXT,
  updated_at TIMESTAMPTZ DEFAULT NOW()
)

referrals (
  id SERIAL PRIMARY KEY,
  company_id UUID REFERENCES companies(id),
  referrer_client_id UUID REFERENCES clients(id),
  referred_name TEXT NOT NULL,
  referred_phone TEXT,
  referred_email TEXT,
  status TEXT DEFAULT 'pending',
  reward_issued BOOLEAN DEFAULT FALSE,
  reward_amount NUMERIC,
  source TEXT DEFAULT 'manual',
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
)

communication_log (
  id SERIAL PRIMARY KEY,
  company_id UUID REFERENCES companies(id),
  client_id UUID REFERENCES clients(id),
  job_id UUID REFERENCES jobs(id) NULL,
  direction TEXT NOT NULL,
  channel TEXT NOT NULL,
  source TEXT NOT NULL,
  sent_by TEXT,
  recipient TEXT,
  subject TEXT,
  body TEXT,
  twilio_message_sid TEXT,
  resend_email_id TEXT,
  delivery_status TEXT DEFAULT 'pending',
  opened_at TIMESTAMPTZ,
  clicked_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
)

communication_events (
  id SERIAL PRIMARY KEY,
  communication_log_id INTEGER REFERENCES communication_log(id),
  event_type TEXT NOT NULL,
  event_data JSONB,
  occurred_at TIMESTAMPTZ NOT NULL
)

All tables: RLS policies scoped to company_id.
Pattern: company_id = (SELECT company_id FROM users WHERE id = auth.uid())

═══════════════════════════════════════════════════════════
AUTOMATIC COMMUNICATION CAPTURE
═══════════════════════════════════════════════════════════

Every outbound SMS via Twilio:
  Insert communication_log row:
    source='system', channel='sms', direction='outbound'
    recipient = client phone, body = full SMS text
    twilio_message_sid = Twilio response SID
    delivery_status = 'sent'

Every outbound email via Resend:
  Insert communication_log row:
    source='system', channel='email', direction='outbound'
    recipient = client email, subject = email subject
    body = plain text version of email body
    resend_email_id = Resend response ID
    delivery_status = 'sent'

Every inbound SMS from Twilio webhook:
  Insert communication_log row:
    direction='inbound', source='system', channel='sms'
    body = inbound message text
    delivery_status = 'delivered'

Every manual message sent from the Message button:
  Insert communication_log row:
    source='staff', sent_by = logged-in user display name
    channel = sms or email, direction = 'outbound'

═══════════════════════════════════════════════════════════
WEBHOOK ENDPOINTS
═══════════════════════════════════════════════════════════

POST /api/communications/sms/status
  Receives Twilio delivery status callbacks
  Updates communication_log.delivery_status
  Inserts communication_events row
  Matches by twilio_message_sid
  Validate using Twilio webhook signature + TWILIO_AUTH_TOKEN

POST /api/communications/email/webhook
  Receives Resend webhook events:
    email.sent, email.delivered, email.opened,
    email.clicked, email.bounced, email.complained
  Updates communication_log delivery_status, opened_at, clicked_at
  Inserts communication_events row for each event
  Matches by resend_email_id
  Validate using RESEND_WEBHOOK_SECRET (add to Replit Secrets)

Both endpoints:
  Return HTTP 200 immediately (async processing)
  No JWT auth required — webhook signature validation only
  Use Supabase service role key for DB writes (bypass RLS)
