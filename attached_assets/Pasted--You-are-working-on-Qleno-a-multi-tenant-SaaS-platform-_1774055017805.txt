You are working on Qleno, a multi-tenant SaaS platform for 
residential and commercial cleaning businesses.

This sprint builds the Commercial Account Architecture — 
the foundational data model that supports property management 
accounts with multiple properties, hourly billing, service-type 
rate cards, multi-contact routing, card-on-file auto-charging, 
and flexible invoicing cadences.

This is a PHES Day 1 requirement. PHES is 70% commercial — 
property management turnovers and common areas — and cannot 
migrate without this architecture in place.

REAL-WORLD CONTEXT — READ BEFORE BUILDING:

PHES operates three commercial billing patterns that must 
all be supported:

PATTERN 1 — On-demand per job, card on file (PRIMARY)
  Client calls or texts requesting a job.
  Office schedules it. Tech goes, clocks in, 
  does the work, clocks out.
  System calculates hours × rate.
  Invoice auto-created.
  Card on file charged immediately on job completion.
  Receipt sent to the billing contact.
  This is the default for PPM accounts like Daniel Walter.
  Example: Danny texts "need turnover at 1000 La Salle" →
  tech clocks in → clocks out after 2.62 hrs →
  $157.20 charged to Daniel's card → done.

PATTERN 2 — Scheduled recurring commercial, card on file
  Commercial client on a fixed recurring schedule.
  Each visit auto-invoices and charges COF on completion.
  Same auto-charge flow as Pattern 1, just scheduled.

PATTERN 3 — NET 30 invoice, no auto-charge (MINORITY)
  Job completes → invoice generated → 
  payment terms apply → client pays by check, 
  ACH, or payment link.
  No auto-charge attempt.
  Owner marks paid manually.
  Standard for Evinco-style commercial contracts.

Almost all PHES clients — residential and commercial — 
pay by card on file. Check/ACH are edge cases.
The system must treat COF as the default path, 
not an optional feature.

TECH STACK: React 18, Tailwind, Express/Node.js, Supabase 
(Postgres + RLS), Plus Jakarta Sans only.

DESIGN SYSTEM — NON-NEGOTIABLE:
- Background: #F7F6F3
- Cards: #FFFFFF
- Primary text: #1A1917
- Secondary text: #6B6860
- Borders: #E5E2DC
- Accent: var(--brand)
- NO emojis, NO dark mode, light theme only
- Plus Jakarta Sans only
- Skeleton loaders for all async loads
- All monetary values: USD, 2 decimal places, 
  comma-formatted
- Status badges: pill shape, muted background, 
  colored text

All endpoints: JWT protected, company_id scoped, 
RLS enforced. Never expose data across tenant boundaries.

PART 3 — HOURLY BILLING ENGINE
═══════════════════════════════════════════════════════

When a job has billing_method = 'hourly'
and is marked complete:

STEP 1 — CALCULATE HOURS:
  Pull all timeclock entries for this job_id
  total_minutes = SUM of 
    (clock_out_time - clock_in_time) in minutes
    across all timeclock entries for this job
  billed_hours = ROUND(total_minutes / 60, 2)
  Example: 2 hrs 37 min = 157 min = 2.62 hrs
  
  Set jobs.billed_hours = billed_hours
  Set jobs.billed_amount = 
    billed_hours × jobs.hourly_rate

STEP 2 — ALLOW MANUAL OVERRIDE:
  Before charging, owner/admin can edit 
  billed_hours on the completed job.
  
  UI shows:
  "Time on site: 2.62 hrs (from clock in/out)"
  "Billed hours: [  2.62  ] hrs" ← editable
  "Billed amount: $157.20" ← updates on change
  
  If billed_hours is edited:
    Recalculate billed_amount automatically
    Log the override in audit_log:
    { action: 'hours_override', 
      original: 2.62, adjusted: 2.50,
      adjusted_by: user_id }

STEP 3 — CREATE INVOICE:
  Auto-create invoice with:
  line_items: [{
    description: service_type + ' — ' + 
                 property_name + ' · ' + 
                 billed_hours + ' hrs @ $' + 
                 hourly_rate + '/hr',
    quantity: billed_hours,
    unit_price: hourly_rate,
    total: billed_amount
  }]
  total_amount: billed_amount
  due_date: today + account.payment_terms_days

STEP 4 — CHARGE CARD OR QUEUE:
  If account.payment_method = 'card_on_file'
  AND account.auto_charge_on_completion = true:

    Check payment provider for this company:
      PHES → Square (square_customer_id)
      All others → Stripe (stripe_customer_id)

    If card on file exists:
      Attempt charge for invoice total_amount
      
      On success:
        Set invoice.status = 'paid'
        Set jobs.charge_succeeded_at = now()
        Set invoice.paid_at = now()
        Send receipt email to all contacts where
        receives_receipts = true
        Toast to office UI: 
        "Job complete — $157.20 charged to 
         Daniel Walter's card on file"

      On failure:
        Set invoice.status = 'sent'
        Set jobs.charge_failed_at = now()
        Set jobs.charge_failure_reason = 
          error message from payment processor
        Surface alert on dashboard 
        Needs Attention queue:
        "Charge failed for [Job] — 
         [failure reason]. Retry or collect 
         payment manually."
        Do NOT retry automatically — 
        owner action required

    If NO card on file:
      Set invoice.status = 'sent'
      Surface alert on account profile 
      AND dashboard Needs Attention queue:
      "No card on file for [Account Name]. 
       Invoice sent but not charged. 
       Send payment link to add card."

  If account.payment_method = 'check' 
  OR 'ach' OR 'invoice_only':
    Set invoice.status = 'sent'
    No charge attempt
    Owner marks paid manually
    No alert surfaced — this is expected behavior

For flat_rate jobs (billing_method = 'flat_rate'):
  billed_amount = jobs.price (existing field)
  Skip hours calculation
  Proceed directly to Step 4 above

PART 5 — CONSOLIDATED INVOICING
═══════════════════════════════════════════════════════

For accounts with invoice_frequency = 
'weekly' or 'monthly':

  Do NOT auto-charge on individual job completion.
  Instead: 
    Auto-create draft invoice per job (for records)
    Set invoice.status = 'pending_consolidation'
    Do NOT charge card yet

  Add POST /api/accounts/:id/generate-invoice
  — Owner/admin triggers manually
  — OR runs automatically on schedule:
    weekly → every Monday for prior week
    monthly → 1st of each month for prior month

  Logic:
    Collect all jobs for this account where
    invoice.status = 'pending_consolidation'
    within the billing period
    
    Create ONE consolidated invoice with 
    one line item per job:
    { 
      description: date + ' · ' + property_name + 
                   ' · ' + service_type + ' · ' +
                   billed_hours + ' hrs @ $' + 
                   hourly_rate + '/hr',
      quantity: billed_hours,
      unit_price: hourly_rate,
      total: billed_amount
    }
    
    Consolidated invoice total = sum of all jobs
    Due date = invoice_date + payment_terms_days
    
    Mark all included job invoices as 
    consolidated_into = new_invoice_id

  Pre-generation preview:
    Before generating, show a preview modal:
    "X jobs · [date range]
     [table of all line items]
     Total: $X,XXX.XX
     [Confirm & Generate] [Cancel]"

  Auto-charge on consolidated invoice:
    Same logic as per-job auto-charge in Part 3.
    If COF → charge total → receipt to 
    contacts where receives_receipts = true.

  On account profile:
    Show uninvoiced jobs count + total when > 0:
    "5 jobs pending invoicing — $847.00"
    Amber indicator, links to Jobs & Invoices tab
