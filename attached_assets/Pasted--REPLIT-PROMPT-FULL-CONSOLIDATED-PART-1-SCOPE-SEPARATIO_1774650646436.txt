**PART 5 — DATABASE**

Run the following migrations:

```sql
-- Jobs table additions
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS upsell_shown BOOLEAN DEFAULT false;
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS upsell_accepted BOOLEAN DEFAULT false;
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS upsell_declined BOOLEAN DEFAULT false;
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS upsell_deferred BOOLEAN DEFAULT false;
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS upsell_cadence_selected VARCHAR(20);
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS property_vacant BOOLEAN DEFAULT false;

-- Remove move_date if it was previously added
ALTER TABLE jobs DROP COLUMN IF EXISTS move_date;

-- Rate locks table
CREATE TABLE IF NOT EXISTS rate_locks (
  id SERIAL PRIMARY KEY,
  company_id INTEGER NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  client_id INTEGER NOT NULL REFERENCES clients(id) ON DELETE CASCADE,
  recurring_schedule_id INTEGER REFERENCES recurring_schedules(id) ON DELETE SET NULL,
  locked_rate NUMERIC(10,2) NOT NULL,
  cadence VARCHAR(20) NOT NULL,
  lock_start_date DATE NOT NULL,
  lock_expires_at DATE NOT NULL,
  active BOOLEAN NOT NULL DEFAULT true,
  voided_at TIMESTAMP,
  void_reason VARCHAR(50),
  -- void_reason values: 'time_overrun' | 'condition_change' | 'service_gap' | 'manual'
  void_notes TEXT,
  voided_by_user_id INTEGER,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_rate_locks_client ON rate_locks(client_id);
CREATE INDEX IF NOT EXISTS idx_rate_locks_active ON rate_locks(active, lock_expires_at);
```

**On upsell accepted — form submission flow:**
1. Create the Deep Clean job with all standard fields plus upsell tracking columns
2. Create a `recurring_schedules` record with the selected cadence, client_id, company_id, service_address, and scope
3. Insert into `rate_locks`:
   - `locked_rate`: full recurring rate (not the discounted first visit rate)
   - `cadence`: selected cadence
   - `lock_start_date`: today
   - `lock_expires_at`: today + 24 months
   - `active`: true
4. Apply 15% discount to the first recurring job only — tag it `first_recurring_discounted: true` on that job record

**PART 6 — RATE LOCK AUTO-VOID LOGIC**

Create `checkRateLockVoidConditions(clientId, companyId)` in `server/utils/rateLock.ts`.

Call this function server-side after every job is marked complete.

**Condition 1 — Time overrun pattern:**
```
Query the last 3 completed recurring jobs for this client
If 2 or more jobs have actual_duration > estimated_duration * 1.20:
  - Set rate_locks.active = false
  - Set void_reason = 'time_overrun'
  - Set voided_at = NOW()
  - Send Resend email to info@phes.io:
    Subject: "Rate Lock Voided — Time Overrun: [Client Name]"
    Body: Client name, locked rate, average actual vs estimated time, prompt to re-quote
```

**Condition 2 — Service gap (run on nightly cron alongside recurring engine):**
```
Query all rate_locks WHERE active = true AND lock_expires_at > TODAY
For each lock, find the most recent completed job for that client
If most_recent_job_date < TODAY - 60 days:
  - Set active = false
  - Set void_reason = 'service_gap'
  - Set voided_at = NOW()
  - Send Resend email to info@phes.io:
    Subject: "Rate Lock Voided — Service Gap: [Client Name]"
    Body: Client name, last service date, days since last service
```

**Condition 3 — Expiry:**
```
Same nightly check: if lock_expires_at < TODAY and active = true:
  - Set active = false
  - Set void_reason = 'expired'
  No email needed for natural expiry
```
