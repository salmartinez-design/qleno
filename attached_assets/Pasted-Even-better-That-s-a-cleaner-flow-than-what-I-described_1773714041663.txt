The Secure Card Link — How It Works
This is called a payment request link. Stripe handles this natively. Here’s the architecture:

You create invoice → 
Stripe generates a unique secure URL → 
URL sent via Resend email + Twilio SMS → 
Client opens link on any device → 
Enters card in Stripe's hosted form → 
Card tokenized and stored against client → 
Invoice marked paid → 
You see it in CleanOps Pro instantly


The link expires after 72 hours for security. Client never creates an account. One tap, card saved, done.

PART B — SECURE PAYMENT LINK

When "Send Card Link via Email" or SMS is clicked:

1. CREATE PAYMENT LINK RECORD
   Table: payment_links
   id, company_id, client_id, 
   token (unique random 32-char string),
   purpose: 'save_card' | 'pay_invoice',
   invoice_id (nullable),
   amount (nullable — for pay_invoice),
   expires_at (timestamptz — 72 hours from now),
   used_at (timestamptz, nullable),
   created_by (user_id)

2. GENERATE SECURE URL
   Format: 
   https://[app-domain]/pay/[token]
   
   This is a PUBLIC route — no login required.
   Anyone with the link can access it.

3. SEND VIA EMAIL (if Resend configured)
   To: client billing email
   Subject: "[Company Name] — Save your 
   payment method"
   Body: clean branded email with:
   - Company name and brand color
   - "Hi [Client Name],"
   - "Please save a payment method for future 
     invoices from [Company Name]."
   - Large CTA button: "Save Payment Method"
     linking to the secure URL
   - "This link expires in 72 hours."
   - "You will not be charged today."

4. SEND VIA SMS (if Twilio configured)
   To: client billing phone or primary phone
   Message: "[Company Name]: Please save your 
   payment method for future invoices. 
   Link expires in 72 hours: [URL]"

5. LOG to notification_log

PART C — PUBLIC PAYMENT PAGE /pay/:token

This page requires NO login.
Accessible by anyone with the link.

On load:
1. Look up token in payment_links table
2. If token not found: show "This link is invalid"
3. If expires_at < now: show "This link has expired.
   Please contact [Company Name] for a new link."
4. If used_at is set: show "This payment method 
   has already been saved. Thank you."

If valid token, show branded page:
- Company logo and brand color at top
- "Secure Payment Method" heading
- If purpose = 'save_card':
  "Save your card for future invoices from 
  [Company Name]. You will not be charged today."
- If purpose = 'pay_invoice':
  "Pay Invoice #[number] — $[amount]"

Stripe Elements card form:
- Card number, expiry, CVC fields
- Cardholder name field
- Checkbox: "Save this card for future invoices"
  (checked by default, cannot uncheck for 
  save_card purpose)
- Submit button: "Save Card Securely" or 
  "Pay $[amount]"
- Stripe badge: "Secured by Stripe"
- "Your card information is encrypted and 
  never stored on our servers."

On successful card save:
1. Create Stripe customer if not exists
   stripe_customer_id stored on client record
2. Attach payment method to Stripe customer
3. Update client record:
   card_last_four, card_brand, card_expiry,
   card_saved_at = now()
4. Mark payment_link.used_at = now()
5. Log to audit_log
6. Show success page:
   Large checkmark SVG (no emoji)
   "Payment method saved successfully."
   "Thank you, [Client Name]."
   "[Company Name] will use this card for 
   future invoices."
   No further action needed — page can be closed.

On failed card:
- Show Stripe error message inline
- Allow retry without generating new link

PART D — AUTO-CHARGE ON INVOICE CREATION

When an invoice is created for a client where
auto_charge = true AND card on file exists:

1. Create invoice record as normal
2. Immediately attempt Stripe charge:
   - Use stored stripe_customer_id
   - Charge invoice amount
3. On success:
   - Update invoice status to 'paid'
   - Create payment record
   - Send receipt email to client:
     Subject: "Receipt from [Company Name] — 
     Invoice #[number]"
     Body: amount charged, last 4 digits,
     date, job details
4. On failure:
   - Keep invoice status as 'sent'
   - Flag: payment_failed = true
   - Send email to client:
     "Your payment of $[amount] was declined.
     Please update your payment method: [new link]"
   - Alert owner in dashboard Needs Attention panel:
     "Payment failed for [Client Name] — $[amount]"
   - Log to notification_log
