FEATURE — GPS GEOFENCING FOR CLOCK IN/OUT

Currently employees can clock in from anywhere. 
Build enforced geofencing so employees must be 
within a set distance of the job address to clock in 
and clock out.

---

PART A — JOB ADDRESS GEOCODING

When a job is created or updated with an address:
- Automatically geocode the address to lat/lng using 
  Google Maps Geocoding API (GOOGLE_MAPS_API_KEY)
- Store lat/lng on the jobs table:
  job_lat (numeric), job_lng (numeric)
- If GOOGLE_MAPS_API_KEY is not configured: skip 
  geocoding, log warning, geofencing will be disabled
- If address cannot be geocoded: flag the job with 
  geocode_failed = true, show warning on job detail

---

PART B — COMPANY GEOFENCING SETTINGS

Add geofencing settings to Company Settings page 
under a new "Clock In/Out" tab:

Settings:
1. Enforce Geofencing (toggle, default ON)
   "Employees must be within range of job address 
   to clock in and out"

2. Clock In Radius (number input, default 500)
   Unit: feet
   "How close must employee be to job address 
   to clock in"
   Range: 100-2640 feet (half mile max)

3. Clock Out Radius (number input, default 1000)  
   "How close must employee be to clock out"
   Slightly larger than clock in to account for 
   employee finishing outside
   Range: 100-2640 feet

4. Override Allowed (toggle, default ON)
   "Allow owner/admin to manually override a 
   failed geofence check"

5. Soft Mode (toggle, default OFF)
   "Warn employee if outside range but still 
   allow clock in — logs the distance violation"

Save button — saves to companies table:
  geofence_enabled (boolean, default true)
  geofence_clockin_radius_ft (int, default 500)
  geofence_clockout_radius_ft (int, default 1000)
  geofence_override_allowed (boolean, default true)
  geofence_soft_mode (boolean, default false)

---

PART C — CLOCK IN ENFORCEMENT (My Jobs / Mobile)

When employee taps "Clock In" on /my-jobs:

1. REQUEST LOCATION
   - Browser requests GPS location
   - If employee denies location: show message
     "Location access is required to clock in. 
     Please enable location in your browser settings."
   - Do not allow clock in without location

2. CALCULATE DISTANCE
   Use Haversine formula to calculate distance 
   between employee's current GPS coordinates 
   and job's geocoded lat/lng.
   
   Haversine formula in JavaScript:


function getDistanceFt(lat1, lng1, lat2, lng2) {
const R = 20902231; // Earth radius in feet
const dLat = (lat2-lat1) * Math.PI/180;
const dLng = (lng2-lng1) * Math.PI/180;
const a = Math.sin(dLat/2) * Math.sin(dLat/2) +
Math.cos(lat1 * Math.PI/180) *
Math.cos(lat2 * Math.PI/180) *
Math.sin(dLng/2) * Math.sin(dLng/2);
const c = 2 * Math.atan2(Math.sqrt(a),
Math.sqrt(1-a));
return R * c;
}


3. GEOFENCE DECISION

If geofence_enabled = false:
- Allow clock in, skip all checks
- Log employee GPS coordinates anyway

If geofence_soft_mode = true:
- Allow clock in regardless of distance
- If outside radius: show amber warning
  "You are X feet from the job address. 
  Your location has been logged."
- Log distance_ft and outside_geofence = true

If distance <= clockin_radius_ft:
- Allow clock in
- Show green confirmation: "Clocked in successfully"
- Log distance_ft, outside_geofence = false

If distance > clockin_radius_ft:
- BLOCK clock in
- Show red message:
  "You are too far from this job location.
  You must be within [X] feet to clock in.
  Current distance: [X] feet.
  Please drive to the job address and try again."
- Log the failed attempt to clock_in_attempts table

4. STORE WITH CLOCK ENTRY
On successful clock in, store to 
time_clock_entries:
- clock_in_lat, clock_in_lng
- clock_in_distance_ft (distance from job)
- clock_in_outside_geofence (boolean)
- clock_in_location_accuracy (GPS accuracy in meters 
  from browser)
