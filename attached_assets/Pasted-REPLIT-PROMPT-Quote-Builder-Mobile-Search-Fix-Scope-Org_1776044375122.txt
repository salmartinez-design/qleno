REPLIT PROMPT — Quote Builder Mobile: Search Fix + Scope Organization + Navigation
Three targeted fixes for the mobile view of /quotes/new. No desktop layout changes.

FIX 1 — MOBILE SEARCH NOT WORKING
The client search input is not responding to taps/typing on mobile. Fix:
	•	Add inputMode="search" and autoComplete="off" to the search input
	•	Ensure the input is not inside any element with pointer-events: none on mobile
	•	The dropdown results panel must render above the keyboard — add position: absolute; z-index: 9999 to the dropdown container
	•	On mobile, after a client is selected, explicitly call input.blur() to dismiss the keyboard
	•	Touch events: ensure onTouchEnd is not being swallowed by a parent scroll container — add onTouchStart={(e) => e.stopPropagation()} to the search input
	•	Test fix: type “jim” on mobile and confirm Jim Schultz appears in results

FIX 3 — MOBILE NAVIGATION: ADD NEXT BUTTON
The bottom bar currently shows only “Estimated Total” and “Save Quote.” This needs a proper Next flow on mobile.
Replace the bottom bar with:
Left side: “Estimated Total” label + price value (or “—” if no scope selected)
Right side — two buttons:
	•	“Save Draft” — ghost style, smaller, 13px
	•	“Next →” — brand color fill, same size as current “Save Quote” button
“Next →” behavior:
	•	If on Step 1 (Customer Info / scope selection): advances to Step 2 (Property Details)
	•	If on Step 2: advances to Step 3 (Add-ons)
	•	If on final step (Review): becomes “Save Quote” primary CTA
“Save Quote” / “Save & Convert to Job” only appears on the final Review step — not on every step.
On mobile the stepper tabs at the top are too small to tap reliably — the Next button at the bottom is the primary navigation. Ensure tapping Next scrolls to the top of the next step automatically (window.scrollTo(0, 0) on step change).
