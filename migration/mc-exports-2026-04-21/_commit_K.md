## Why

The 272 rows the J5 manual trigger generated landed on the wrong dates. The
diagnostic ("Dispatch shows 25/$5,962 on Apr 23, MC says 14/$2,900") traced
the clustering to **78 of 79 recurring_schedules having NULL `day_of_week`
and `start_date = '2026-03-23'`** (the B-series import default). The engine
code is correct given its inputs — the bug is in the data. `generateOccurrences`
falls back to `start.getDay()` when `day_of_week` is NULL, so every null-DOW
schedule clusters on Mondays (start_date 2026-03-23 = Monday) or on the 23rd
of each month.

We're about to import MC's dispatch export as the source of truth for
2026-01-01 → 2026-04-30, so the engine-generated future rows must be gone
before the import lands. Wide-scope rollback: delete ALL engine-generated
scheduled rows from 2026-04-01 onward.
