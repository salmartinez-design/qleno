# Qleno Native App (Capacitor) â€” Setup & Release

Phase 1 of the App Store launch: wrap the existing Vite web app in a Capacitor
shell for iOS and Android. The web bundle (`artifacts/qleno/dist/public`) is
bundled into the native app; API calls hit the production Railway API.

> **Why bundled assets (not a remote webview)?** Apple rejects apps that are
> "just a website" (App Store Review Guideline 4.2 â€” minimum functionality).
> We ship the real web bundle inside the app and only the API traffic goes
> remote, so it reads as a native app with native push.

## What's in the repo (this phase)

| File | Purpose |
|------|---------|
| `artifacts/qleno/capacitor.config.ts` | App id `io.phes.qleno`, name `Qleno`, `webDir: dist/public`, push + splash config |

The `ios/` and `android/` native projects are **not** committed yet â€” they are
generated by `npx cap add` on a machine with the platform SDKs (see below).
iOS specifically **requires macOS + Xcode** and cannot be generated on the
Linux CI/dev container.

## How API calls work in the shell

Every call site uses bare relative URLs (`fetch('/api/...')`). On the web those
resolve against the SPA's origin. In the native shell the origin is
`capacitor://localhost` (iOS) / `http://localhost` (Android), so
`native-bridge.ts` rewrites any path starting with `/` to:

```
VITE_NATIVE_API_ORIGIN  (default https://workspaceapi-server-production-b9d4.up.railway.app)
```

Auth already travels as a Bearer token in the `Authorization` header
(`getAuthHeaders()`), so it survives the cross-origin hop â€” no cookie reliance.

**API follow-ups for the shell (not blocking the wrapper):**
1. The API must send permissive CORS for the capacitor origins on the routes
   the app hits (`Access-Control-Allow-Origin` for `capacitor://localhost` and
   `http://localhost`, allow `Authorization`).
2. Three call sites use `credentials: 'include'` (cookie auth):
   `auth/refresh`, `auth/logout`, `leave-review`. They authenticate fine via
   the Bearer token but should be confirmed against the native origin.
3. Any `<img src="/uploads/...">` style relative asset references also need the
   origin prefix (the fetch shim only covers `fetch`).

## First-time native project generation (on macOS)

```bash
cd artifacts/qleno
pnpm install
pnpm run cap:build          # production web build into dist/public
pnpm exec cap add ios        # macOS + Xcode required
pnpm exec cap add android    # Android Studio + SDK required
pnpm exec cap sync
```

Commit the generated `ios/` and `android/` folders (Capacitor writes their own
`.gitignore` for build artifacts/Pods).

## Day-to-day: ship a web change to the apps

```bash
cd artifacts/qleno
pnpm run cap:sync           # rebuilds web + copies into native projects
pnpm run cap:ios            # opens Xcode  â†’ Product > Archive â†’ TestFlight
pnpm run cap:android        # opens Android Studio â†’ Build > Generate Signed Bundle
```

## TestFlight (iOS)

1. Apple Developer account + an App Store Connect app record for `io.phes.qleno`.
2. In Xcode: set the Team/signing, bump build number, **Product > Archive**.
3. Distribute App â†’ App Store Connect â†’ Upload.
4. The build appears in TestFlight after processing; add internal testers.
